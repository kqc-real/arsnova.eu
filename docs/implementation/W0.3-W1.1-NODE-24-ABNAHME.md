# W0.3 / W1.1 – Node-24-Abnahme

**Stand:** 2026-07-24  
**Referenzversion:** Node.js 24.18.0 LTS („Krypton“)

## Umfang

- `.nvmrc` pinnt Node 24.18.0 als lokalen Referenzpfad.
- Root-`engines` unterstützt Node 24 und den CI-Kompatibilitätspfad Node 22.12+; Node 20 ist entfernt.
- Docker-Builder und Produktionsstufe verwenden `node:24-alpine`.
- Build- und Typecheck-Matrix prüft Node 22 und 24.
- Alle übrigen CI-, Landing- und Artefaktjobs verwenden Node 24 als Referenzruntime.
- Entwickler-, CI- und Sicherheitsdokumentation beschreibt denselben Versionsstand.

## Betriebsfolge

Nach dem Merge baut die Deployment-Pipeline das Produktionsimage auf Node 24 neu. Es sind
keine neuen Environment-Variablen oder Datenbankmigrationen erforderlich. Der reguläre
Post-Deploy-Smoke bleibt das Freigabegate; ein Rollback erfolgt über das bestehende
Deployment-Rollback auf das vorherige Image.
